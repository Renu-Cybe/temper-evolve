# Day 1 — Temper v2.0 Codong 风格重构

**Time:** 2026-03-28
**Version:** v2.0
**Change:** 全新架构

---

## What

从 v0.x 的单文件脚本，重构为模块化架构，借鉴 [Codong](https://github.com/brettinhere/Codong) 的结构化错误设计。

### 新架构

```
temper/
├── core/
│   ├── errors.py          # 结构化错误系统
│   └── types.py           # 类型验证
└── tools/
    ├── fs.py              # 文件系统模块
    └── shell.py           # 命令执行模块
```

### 核心改进

| 特性 | 旧版 (v0.x) | 新版 (v2.0) |
|------|-------------|-------------|
| 错误格式 | 字符串 `"Error: ..."` | 结构化 `{"error": "E1001", "message": "...", "fix": "..."}` |
| 工具命名 | `read_file`, `run_bash` | `fs.read`, `fs.write`, `fs.edit`, `shell.run` |
| 模块组织 | 平铺 | 分层（core/tools） |
| 类型检查 | 无 | 自动验证 |

### 错误系统设计（学习 Codong）

```python
# 错误包含 code, message, fix, retry 四个字段
{
    "error": "E1001_FILE_NOT_FOUND",
    "message": "文件不存在: temper.py",
    "fix": "检查路径是否正确，或使用 fs.exists() 验证",
    "retry": False,
    "ok": False
}
```

这让模型能：
1. 理解错误原因（message）
2. 知道如何修复（fix）
3. 判断是否可以重试（retry）

---

## Why

原架构的问题：
1. **错误信息不透明** - 模型看不懂字符串错误
2. **工具难以扩展** - 所有工具挤在一个文件
3. **无类型安全** - 参数错误只能在运行时暴露

借鉴 Codong 的设计理念：
- **单一原则** - 每个工具只做一件事
- **结构化错误** - 机器可读的错误格式
- **显式优于隐式** - 工具名带命名空间（fs.read 而非 read_file）

## 优化

### format_result 智能截断（迭代优化）

**问题：** 读取长文件时内容被截断到 2000 字符，模型无法看到完整代码。

**解决方案：** 优化 `format_result` 函数：

1. **参数化最大长度** - `max_length=8000`，可配置
2. **智能字符串截断** - 尝试在换行处截断，保持可读性
3. **复杂类型截断** - 列表/字典超过阈值时保留前 N 个并提示剩余数量
4. **显示原始大小** - 截断时显示总字符数，让模型了解完整规模

```python
def format_result(result, max_length=8000):
    """格式化工具结果，智能处理过长内容"""
    if isinstance(value, str) and len(value) > max_length:
        # 尝试在换行处截断
        truncated = value[:max_length]
        last_newline = truncated.rfind('\n')
        if last_newline > max_length * 0.8:
            truncated = truncated[:last_newline]
        value = truncated + f"\n... [内容过长，已截断，共 {len(value)} 字符]"
```

**效果：** 现在读取 `temper.py` 可以看到完整代码，不再被截断。

## 对话记忆系统（同 Day 1 下午）

**问题：** 每次对话后 `messages` 被重置，模型无法记住之前的上下文。

**例如：**
```
👤 你: 读取 README.md
🤖 Temper: {...读取结果...}

👤 你: 总结一下刚才的内容
🤖 Temper: 什么内容？我不知道你在说什么...
```

**解决方案：** 实现全局对话记忆

### 设计

```python
# 全局对话历史
conversation_history = []
MAX_HISTORY = 20  # 保留最近 20 轮

def get_messages_with_history(current_input):
    """构建包含历史的 messages"""
    messages = [{"role": "system", "content": SYSTEM_PROMPT}]

    # 添加最近历史
    if conversation_history:
        recent = conversation_history[-MAX_HISTORY*2:]
        messages.extend(recent)

    messages.append({"role": "user", "content": current_input})
    return messages

def add_to_history(user_input, assistant_response):
    """保存对话到历史"""
    conversation_history.append({"role": "user", "content": user_input})
    conversation_history.append({"role": "assistant", "content": assistant_response})

    # 自动清理旧记录
    while len(conversation_history) > MAX_HISTORY * 2:
        conversation_history.pop(0)
```

### 新增命令

| 命令 | 功能 |
|------|------|
| `/clear` | 清空对话历史 |
| `/history` | 查看当前历史长度 |

### 效果

```
👤 你: 读取 README.md
🤖 Temper: {...读取结果...}

👤 你: 总结一下刚才的内容      ← 现在有上下文！
🤖 Temper: 这是 Temper v2.0 的说明文档，特性包括...

👤 你: /history
📜 当前对话历史: 2 轮 (最多保留 20 轮)
```
